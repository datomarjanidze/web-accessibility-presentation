Keyboard Navigation

<button tabindex="-1">tab is disabled</button>
<button tabindex="1" class="no-focus-indicator"> Is tabbable but has no focus indicator </button>
<button tabindex="3">And then this</button>
<button tabindex="2">This will be focused first</button>
<button (click)="openPopup('bad-example')">Open popup</button>
  1. The first button has a tabindex "-1", so the users won't be able to focus on that button and the users with screen readers will not have any chance to click that button
  2. The second button is tabbable but has no visual indicator that it is focused, so the user will be able to focus on that button but will not see if it is focused
  3. The third and fourth buttons have wrong tab indices, so the first one which will get the focus will be the fourth one, and then the third one
  4. The auto focus on the popup element is disabled, so the user won't know where the focus is
  5. To avoid keyboard traps, do not disable closing the popup with the escape button, or else users with reading disabilities won't be able to close the popup with the keyboard
<button tabindex="2">This will be focused first</button>
<button tabindex="3">And then this</button>
<button (click)="openPopup('good-example')">Open popup</button>
  1. There should not be any button with tabindex -1 if it does not serve some exceptional technical purpose in the application
  2. There should not be any button without a visual indicator when it is focused
